基本概念

语言特点

Go语言保证了既能到达静态编译语言的安全和性能，又达到了动态语言开发维护的高效率，使用一个表达式来形容Go语言:Go = C+ Python,说明Go语言既有c静态语言程序的运行速度，又能达到Python动态语言的快速开发。

  • 从c语言中继承了很多理念，包括表达式语法，控制结构，基础数据类型，调用参数传值，指针等等，也保留了和c语言一样的编译执行方式及弱化的指针。
    引
  • 入包的概念，用于组织程序结构，Go语言的一个文件都要归属于一个包，而不能单独存在。
  • 垃圾回收机制，内存自动回收，不需开发人员管理
  • 天然并发(重要特点)
    • 从语言层面支持并发，实现简单
    • goroutine轻量级线程，可实现大并发处理，高效利用多核
    • 基于CPS并发模型实现
  • 吸收了管道通信机制，形成go语言特有的通道channel。通过管道channel，可以实现不同的goroute之间的相互通信
  • 函数可以返回多个值
  • 新的创新：切片 延时执行defer

编译和执行

  • 有了go源文件，通过编译器将其编译成机器可以识别的二进制码文件
  • 在该源文件目录下，通过go build对 hello.go文件进行编译。可以指定生成的可执行文件名，在windows下必须是.exe后缀。
    • go build -o myhello.exe hello.go
  • 如果程序没有任何错误就没有任何提示，会在当前目录下出现一个可执行文件，该文件是二进制文件。如果程序有错误 编译时，会在错误的那行报错
  • 也可以使用go run指令 直接执行源码。这种方式需要在go环境下执行，可执行文件的执行不需要go环境但是编译器会将程序运行的依赖的库文件包含在可执行文件中，所以，可执行文件变大了很多

注意事项

go编译器是对文件一行一行进行编译的 所以不要把多条语句写在同一行

go语言定义的变量或者import的包如果没有使用到 不能通过编译

注释与java一样

转义符：\t：制表符，\n：换行符，\r：回车

其他数值型

字符

字符串:go语言的编码都统一成了utf-8 没有乱码问题

字符串类型是不可变的

如果x想修改字符串可以转换成byte 和rune

字符串的两种表现形式

  • 双引号 会识别转义字符
  • 反引号：以字符串的原生形式输出 包括换行和特殊字符 可以实现防止攻击 输出源代码等效果

当一个字符串太长 可以分行写 但是注意 +号要留在上一行

基本数据类型的默认值:整形和浮点型为0 字符串为”” 布尔类型为false

基本数据类型的相互转换

数据类型转换可以从大到小 也可以从小到大 但是从大到小转换的时候会做溢出处理

被转换的是变量存储的数据 变量本身的数据类型没有变化

函数

如果函数返回多个值，在接收时希望忽略某个返回值使用_符号

函数的基本类型和数组都是值传递:如果希望传入函数内的变量能修改函数外的变量可以传入变量的地址&,函数内以指针的方式操作变量

init函数

每一个源文件都可以包含一个init函数，该函数会在main函数执行前，被Go运行框架调用，也就是说init会在main函数前被调用。

如果一个文件同时包含全局变量定义，init函数和main函数 则执行流程是全局变量定义-> init -> main

面试题：如果main目录下的main.go和utils目录下的utils.go都含有变量定义 init函数 那么执行流程是？

异常处理

GO中处理异常的方式：defer，panic，recover

go中可以抛出一个panic的异常 然后在defer中通过recover不会这个异常 然后正常处理

go中也支持自定义错误 ，使用errors.New(返回一个error类型的值，表示一个错误)和panic内置函数(接受一个interface{}类型的值作为参数，可以接收error类型的变量，输出错误信息，并退出程序)

同一个包下不能有相同的函数名

数组

切片

切片是数组的一个引用，因此切片是引用类型，在进行传递时，遵守引用传递的机制。切片的使用和数组类似，遍历切片、访问切片的元素和求切片长度len(slice)都一样。切片的长度是可以变化的，因此切片是一个可以动态变化数组

slice是一个引用类型 底层是一个结构体 对切片的修改 会改变数组

通过make创建切片：var 切片名 []type = make([]type,len,[cap])

定义切片的时候指定具体数组

切片的遍历与数组的遍历相似 切片可以继续切片

使用append函数可以对切片进行动态追加

继承

结构体可以使用嵌套匿名结构体所有的字段和方法

结构体如果嵌入多个匿名结构体 且两个匿名结构体有相同的字段和方法 且结构体本身没有 在访问时必须指定匿名结构体名字否则编译报错

如果一个struct嵌入了一个又名结构体 这种模式是组合 那么在范根组合的结构体的字段或方法时 必须带上结构体的名字

嵌套匿名结构体后，可以在创建结构体变量时，直接指定各个匿名结构体字段的值

goroutine和channel

goroutine

一个go线程上可以起多个协程，写成是轻量级的线程

go协程的特点

  • 有独立的栈空间
  • 共享程序堆空间
  • 调度由用户控制
  • 协程是轻量级的线程

设置cpu

channel

channle本质是一个队列 是线程安全的多个goroutine访问时不需要加锁

channel是有类型的 一个类型的channel只能存放该类型的数据

channel是引用类型 且必须make之后才能使用

channel的关闭:使用内置函数close管理channel 关闭后不能像channel中写入数据 但是仍然可以从channel中读取数据

channel的遍历：遍历时如果没有关闭channel会报错

声明管道为只读或只写

select解决从管道读取数据阻塞的问题

可以在goroutine中使用recover解决写成中出现panic导致程序崩溃的问题

反射

reflect.TyepOf()：获取变量的类型 返回reflect.Type类型

reflect.ValueOf()：获取变量的值 ，返回reflect.Value类型 (是一个结构体 可以获取到关于该变量的很多信息)

变量，interface{}，reflect.Value可以相互转换

反射的注意事项和细节

  • reflect.Value.Kind：获取变量的类型 返回一个常量
  • Type和Kind的区别：Type是类型，Kind是类型 两者又是相同 比如int类型，有时不同 比如type Cat struct 。kind返回struct，Type返回Cat

通过反射来修改变量

reflect.Value.Elem()：Elem returns the value that the interface v contains or that the pointer v points to.
