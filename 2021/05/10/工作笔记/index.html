week1

day01

修改代码并上传

把代码clone到linux /home/bingjiang.bao目录下 进行修改

修改代码之后 使用git flow 上传

git flow init

git flow feature start timeLogFeature

git flow feature publish timeLogFeature

提交到远程仓库 提交compare给群里的成员review

没有问题之后结束feature：git flow feature finish timeLogFeature

提交到develop：git push origin develop

在gitlab上提交merge申请

上传配置

登录atlantis 进入负责的模块

在linux上clone配置文件仓库

在仓库目录下 上传项目中的配置

并上传到配置仓库

之后在配置管理中打tag并编辑配置

上线

merge请求被通过之后 会触发钩子函数 发送请求到jenkins做一些事情比如从github上下载文件 执行build.sh等事情

进行上线

添加进程：添加一台服务器

可以对进程进行分组管理

在上线管理中进行上线

准备上线 -> 选择要上线的服务

添加配置标签和版本号 上线前动作没意义 上线重启

day02

熟悉rta_stats_sync模块

在linux搭建运行环境

编辑.bash_profile 配置运行环境

执行source .bash_profile 刷新配置

拷贝.m2文件 用于maven

创建target目录

执行build.sh skip

生成一条软链 ln -s target/RTBAdinfo-1.73.2 current

从shell可以看到删除了当前文件夹下的配置文件 所以要拷贝linux中tmp下的配置文件到当前目录

再拷贝tmp下的RTBAdinfo.sh到当前目录

这个shell中通过软链启动项目 执行命令./RTBAdInfo.sh start 真正执行的是RTBAdinfo-1.73.2/bin/RTBAdinfo

学习rta_Adinfo代码
